项目概述
开发一个基于Python的WebShell流量检测分析工具，具备实时监控与离线分析双重能力，融合特征检测、行为分析、解密还原与AI智能研判，提供可视化的威胁评估与报告输出。

一、流量采集与输入模块
功能要求
双模式数据源

实时抓包：基于Scapy或pcapy监听指定网卡，支持多网卡选择

离线分析：支持上传PCAP/PCAPNG文件，支持大文件分块读取（按流/按包分批处理）

BPF过滤

用户可自定义BPF语法过滤规则（如 port 80 or 443，host 192.168.1.1）

实时模式与离线模式均支持过滤

大文件处理

分块解析（按包索引/按会话分批加载）

支持流量采样率配置（如每100个包采样1个），防止内存溢出

二、WebShell行为检测引擎
2.1 特征匹配层
指纹库内置以下工具流量特征：

菜刀、蚁剑、冰蝎（Behinder）、哥斯拉（Godzilla）、Weevely3、SharPyShell

检测维度：

User-Agent异常（非典型浏览器UA、固定UA模式）

请求头固定组合（如特定顺序的Accept、Connection等）

URL路径特征（含敏感词汇、伪静态后门路径）

Payload编码特征：Base64/Hex/URL编码

危险函数调用：eval、system、assert、exec、preg_replace 等

2.2 协议解析层
支持：

HTTP/HTTPS（允许用户提供TLS密钥或配置代理解密）

DNS隧道（解析TXT、MX、CNAME等记录中的外发数据）

TCP/UDP会话（自定义协议或简单C2）

提取字段：

请求方法、URI、POST Body、Cookie、响应状态码、响应体

对编码字段自动尝试解码（Base64/URL/Hex）

2.3 行为分析层
检测异常模式：

非工作时段（如02:00–06:00）高频短连接

固定心跳间隔（如每60秒一次精确请求）

POST请求体大小异常（极大或极小）

静态文件类型（.jpg/.ico/.css）突然接收POST参数

可执行脚本文件（.php/.jsp/.asp/.aspx）被频繁访问且带有命令参数

2.4 熵值与加密检测
计算请求/响应体的信息熵（Shannon熵）

高熵流量（阈值可配置，如熵 > 6.5）标记为可疑加密通信

结合载荷长度稳定性、填充特征（如AES块对齐）提升检测准确率

三、解密与载荷还原模块
3.1 内置解密能力
冰蝎：AES-128-ECB/XOR 解密（支持不同版本的密钥提取方式）

哥斯拉：支持自定义加密算法（Java版/PHP版）

蚁剑：Base64/ROT13/编码后的命令还原

Weevely3：参数混淆还原

3.2 可扩展机制
用户可通过配置文件或Python接口注册自定义解密函数

支持对新变种或私有加密算法进行插件化适配

3.3 还原内容输出
解密后展示攻击者的：

执行的系统命令

文件操作（上传/下载/读取路径与内容）

数据库查询SQL

内网探测指令

四、AI智能分析模块（核心增强）
4.1 流量包智能选中
在Web/GUI界面以表格/列表形式展示流量包，每行带复选框

支持框选（Shift/Ctrl批量选择）连续或多个流量包

一键触发AI分析

4.2 上下文自动构建
选中流量包后，系统自动拼接以下上下文发送给AI：

完整请求/响应头

Body内容（若为编码则自动解码）

TCP流重组后的会话上下文

五元组（源IP、源端口、目的IP、目的端口、协议）

时间戳及偏移

前后3–5个关联包摘要（请求–响应配对）

4.3 AI接口要求
统一API适配层：支持 OpenAI GPT-4 / Claude / 通义千问 / 文心一言

可在配置文件中切换模型、API地址、密钥

支持通过代理访问外网API

Prompt结构固定要求包含：

text
判断是否为WebShell通信流量及置信度（0–1）
识别具体工具类型（菜刀/蚁剑/冰蝎/哥斯拉/自定义/未知）
分析攻击意图（命令执行/文件上传/数据库操作/权限维持/内网扫描）
提取关键攻击载荷和执行的命令原文
给出处置建议和威胁等级（高/中/低）
4.4 分析结果可视化
以卡片形式展示在选中流量包旁

内容包含：

威胁判定图标 + 工具标识

攻击摘要（一句话总结）

关键命令高亮显示

置信度评分 + 威胁等级

支持将单次或多次AI分析结果导出为分析报告附件（Markdown/HTML）

4.5 批量AI分析
支持对筛选后的可疑流量列表进行批量AI分析

异步并发请求（可配置最大并发数 + 速率限制，防止API限流）

生成汇总分析报表，包含命中统计、高频工具、典型命令、威胁分布

4.6 本地模型支持（可选）
支持接入本地部署的开源模型：Qwen2.5、Llama3、DeepSeek-Coder

通过 Ollama 或 vLLM 提供本地推理接口

流量敏感数据可完全“不出域”分析

五、威胁评估与告警输出
5.1 综合风险评分（0–100）
评分权重建议：

特征命中数量（0–30分）

行为异常度（0–25分）

加密强度/熵值（0–15分）

AI分析置信度（0–30分）

5.2 结构化输出格式
每条检测会话输出包含：

text
- 五元组
- 时间戳（起止）
- URL/请求路径
- 请求/响应摘要（前200字符）
- 解密后的载荷内容
- AI分析结论摘要
- 风险等级（高/中/低）
- 建议处置措施（阻断/隔离/仅告警/忽略）
5.3 报告导出
支持导出格式：

JSONL（每行一个会话）

CSV / Excel

HTML 报告（带排序、搜索、筛选）

可视化时间线图表（展示攻击链）

六、架构与性能要求
6.1 模块化架构
各模块完全解耦：

抓包引擎模块

协议解析器模块

检测引擎模块

解密插件模块

AI分析模块

输出模块

6.2 异步处理
AI分析模块基于 Celery 或 RQ 异步队列

不阻塞主抓包与解析流程

6.3 高性能IO
实时监听模式下，支持多线程/异步IO（asyncio + multiprocessing）

保证低延迟、高吞吐（千兆网络不丢包）

6.4 Web管理界面
后端：FastAPI

前端：Vue3 或 React

功能包括：

抓包控制（启动/停止/过滤配置）

流量列表与详情查看

批量AI分析

报告下载与历史记录

6.5 配置文件
使用YAML格式

可配置项：

AI API密钥、模型名称、Endpoint

分析Prompt模板（支持自定义）

代理设置（HTTP/HTTPS）

评分权重参数

熵值阈值、心跳间隔范围等行为参数

输出要求
请将上述需求作为系统开发的设计依据，按模块输出实现，确保高内聚低耦合、可扩展、便于运维与二次开发。
